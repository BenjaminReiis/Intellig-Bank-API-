02 · segurança

Do login ao acesso autorizado

Toda rota sensível exige um token válido — sem exceções.

passo 1

Cadastro

Senha é convertida em hash com Bcrypt antes de ser persistida.

passo 2

Login

Credenciais validadas contra o hash armazenado.

passo 3

Emissão do JWT

Token assinado é devolvido ao cliente com tempo de expiração.

passo 4

Bearer Token

Toda requisição subsequente envia o token no header Authorization.

Authorization: Bearer <token> — exigido em todas as rotas de conta e operações

app/routers/transactions.py
@router.post("/transfer", response_model=TransactionOut)
def transfer(
    payload: TransferRequest,
    current_user: User = Depends(get_current_user),
    db: Session = Depends(get_db),
):
    validate_balance(payload.origin_account_id, payload.amount, db)
    transaction = execute_transfer(
        db=db,
        origin_id=payload.origin_account_id,
        destination_id=payload.destination_account_id,
        amount=payload.amount,
    )
    return transaction
